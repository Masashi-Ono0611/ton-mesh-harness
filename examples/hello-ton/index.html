Hello, .ton

This page is hosted on TON Storage and addressed via TON DNS — no AWS, no Cloudflare, no domain registrar that can take it down.

If you can read this, the bag has propagated and the DNS record points at it.

How it got here

npx -y ton-sovereign-deploy ./examples/hello-ton \
  --domain <yours>.ton \
  --no-watch

That's the whole deploy. The CLI uploads the directory to TON Storage, signs a change_dns_record transaction via your wallet, and waits until TONAPI confirms the record is live.

What you can do with it

  • Replace this index.html with your actual app and re-run the command.
  • Use --watch so changes auto-redeploy and your machine seeds the bag.
  • Use --wallet-mode agentic in CI to push from GitHub Actions without a phone.

Built with ton-sovereign-deploy. A bag id never changes for the same content — immutable by design.
