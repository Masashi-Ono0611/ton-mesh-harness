# `docs/archive/` — historical, point-in-time records

These docs describe a **past state** of the project (prior-version roadmaps,
research probes, design memos, postmortems). They're kept for design
rationale and provenance, **not** maintained against the shipped product —
read them as history, not as current behaviour. For what's current, see
[`../README.md`](../README.md) (the docs index) and the
[dashboard](../dashboard.html).

Cross-links between archived docs are preserved (they were moved together).
Inbound links from current docs / source comments were updated to point
here on the move.
