ton-projects/sovereign-deploy-kit · CLI + MCP server

Sovereign Deploy Kit

The frontend can’t be taken down
if there’s no server to take down.

One command ships any static build to TON Storage and a .ton domain — zero infra setup. The npx-style CLI for TON’s digital-resistance stack, built for engineers.

01  Concept

One command, TON’s native infra wired together

Detect the build dir, auto-install storage-daemon, upload the bag, point .ton DNS at it, and seed it from your own machine — from the CLI, as a library, or via MCP.

The visitor never talks to a centralized server — only TON gateways or a .ton resolver.
02  Problem

DeFi front-ends keep getting silenced

The contract lives on; the website dies — because it sits on ordinary servers and ordinary DNS, two seizable layers in front of an unstoppable contract. Tornado Cash, Uniswap, 1inch, Balancer all lost the frontend, never the chain.

Same site, two delivery stacks — only the sovereign one survives a takedown order.
03  Solution

Self-host first, signed end-to-end

TON already had Storage, on-chain DNS, and ADNL — this kit is the missing one-command wrapper. The default mode is self-host (--watch): your machine seeds the bag, exactly how foundation.ton runs.

One command wires the whole stack — and an agent can drive it end-to-end.
CLI
What: one command in your terminal. Use it to ship a build to .ton by hand.
MCP
What: a 3-tool server. Use it to let an AI agent run the whole deploy, no human in the loop.
SDK
What: a TypeScript import. Use it to wire sovereign deploys into your own pipeline.

v0.1 → v0.9.0, shipped
